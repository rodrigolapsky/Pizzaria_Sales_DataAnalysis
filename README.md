## 🍕 Pizza Sales & Revenue Data Analysis
This repository contains an end-to-end data analysis project focused on Pizza Restaurant Management. The project covers the entire data lifecycle, from initial data cleaning and exploratory analysis (EDA) using Python to the creation of a high-level strategic dashboard in Power BI.

## 🚀 Business Problem
The objective of this analysis is to monitor the pizzeria's sales performance, identify the most profitable products, understand customer preferences regarding sizes and categories, and optimize decision-making for marketing and inventory control.

## 🛠️ Technologies & Tools
Python (Jupyter Notebook): Data cleaning, decimal separator handling (point/comma correction), missing value treatment, and data type conversion (float64, datetime).
Pandas: For data manipulation, revenue aggregation, and creation of calculated columns.
Matplotlib & Seaborn: For exploratory data analysis (EDA) and price outlier identification.
Power BI: To build an interactive dashboard with custom DAX measures and dynamic visualizations.

### 1. Sales Performance
Strategic KPIs: Real-time monitoring of Total Revenue, Profit, and Average Ticket.
Time Series: Monthly revenue trend analysis using order_date conversion.
Product Ranking: Detailed breakdown of the Top 20 best-selling flavors with percentage (%) share.

### 2. Product Mix & Preferences
Size Distribution: Volumetric sales analysis by size (L, M, S, XL, XXL).
Categorization: Performance segmentation by pizza type (Classic, Chicken, Supreme, Veggie).
Interactive Slicers: Data slicers that allow switching the view by product category instantly.

### 3. Customer Behavior & Profitability
Average Ticket: Dynamic calculation to understand the average spend per transaction.
Margin Analysis: Identification of the flavors that contribute most to the final operation profit.
Consumption Patterns: Identifying purchase trends to create optimized combos and promotions.
